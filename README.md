# 3week_codex

1. 먼저 개념을 정확히 잡기

미니 Redis는 그냥 dict 하나가 아닙니다.
정확히는 dict를 내부 저장소로 쓰는 작은 데이터 서버입니다.
즉 핵심은:
dict = 데이터를 담는 엔진
미니 Redis = dict + 명령어 처리 + 서버 + TTL + 저장/복구
2. 가장 먼저 만들 내부 저장소

store = {}
실제 key-value 저장
expire_at = {}
TTL 만료 시간 저장
이 두 개가 미니 Redis의 핵심 메모리입니다.
3. 기본 명령어부터 구현

SET key value
GET key
DEL key
EXISTS key
INCR key
여기까지 되면 “저장소”처럼 보이기 시작합니다.
4. TTL 기능 추가

SETEX key seconds value
GET 하거나 EXISTS 할 때 먼저 만료됐는지 검사
만료됐으면:
store에서 삭제
expire_at에서도 삭제
이 단계가 들어가면 “인증번호/세션 만료” 같은 실제 서비스 느낌이 납니다.
5. 명령어를 해석하는 부분 만들기

예: "SET name donghyun" 입력
내부적으로는:
명령어 분리
어떤 함수 실행할지 결정
결과 반환
즉:
파서
실행기
이 둘이 필요합니다.
6. 서버 형태로 만들기

여기서부터 진짜 “Redis처럼” 됩니다.
socket 서버를 열고 요청을 받습니다.
클라이언트가 명령을 보내면:
서버가 명령을 파싱
store/expire_at에 반영
결과 응답
이때 네가 만든 프로그램 자체가 “미니 Redis 서버”가 됩니다.
7. 동시에 여러 요청이 와도 안전하게 만들기

조회수 증가 같은 건 동시에 들어오면 꼬일 수 있습니다.
그래서 Lock 같은 걸 써서 보호하는 게 좋습니다.
특히 INCR, DEL, SETEX 같은 변경 작업은 중요합니다.
8. 저장 기능이 필요하면 파일 저장/복구 추가

기본 dict는 프로그램 끄면 날아갑니다.
저장되게 하려면:
서버 시작 시 저장 파일 읽기
실행 중 주기적으로 저장하거나 변경 시 저장
보통:
store
expire_at
둘 다 파일에 저장해야 합니다.
9. 시작할 때의 흐름

서버 시작
저장 파일 있는지 확인
있으면 읽어서 store, expire_at 복구
없으면 빈 상태로 시작
그 다음 요청 대기

11. 게시글 웹페이지와 연결할 때 구조
이건 보통 역할이 3개입니다.

14. 한 줄로 최종 정리

미니 Redis는 딕셔너리처럼 보이지만, 실제로는
딕셔너리를 내부 엔진으로 쓰는 작은 데이터 서버
그리고 구현 순서는
저장소 -> 명령어 -> TTL -> 서버 -> 저장/복구 -> 웹 연결
이렇게 가면 됩니다.



15. 가장 안 꼬이는 개발 순서는 이렇습니다.

메모리 저장소부터 만든다
store = {} 와 expire_at = {} 를 만들고, 클래스 하나로 묶습니다.
이 단계 목표는 “데이터를 메모리에 넣고 꺼낼 수 있다”입니다.

기본 명령어를 구현한다
SET, GET, DEL, EXISTS, INCR 를 먼저 만듭니다.
여기까지 되면 미니 Redis의 뼈대가 생깁니다.

TTL 기능을 붙인다
SETEX 와 만료 검사 함수를 만듭니다.
GET/EXISTS/INCR 전에 만료 여부를 확인하도록 붙입니다.

저장/복구 기능을 붙인다
서버 시작 시 파일을 읽어 store, expire_at 를 복구하고,
데이터가 바뀔 때 파일에 저장합니다.
처음엔 JSON 하나로 저장하는 방식이 제일 쉽습니다.

로컬 테스트를 먼저 끝낸다
네트워크 붙이기 전에 파이썬 내부에서 함수 호출로 확인합니다.
예: SET -> GET, SETEX -> 몇 초 후 GET, 저장 후 재시작 -> 복구.

명령어 파서를 만든다
"SET name donghyun" 같은 문자열을 받아서 명령과 인자를 분리합니다.
그리고 어떤 함수(set, get, incr)를 실행할지 연결합니다.

서버(socket)를 붙인다
이제 네 프로그램이 진짜 “미니 Redis 서버”가 됩니다.
클라이언트 요청을 받고, 파싱하고, 실행하고, 결과를 돌려줍니다.

동시성 보호를 넣는다
여러 요청이 동시에 들어올 수 있으니 Lock 을 붙입니다.
특히 INCR, DEL, SETEX, 저장 함수는 보호하는 게 좋습니다.

게시판 웹 서버를 만든다
별도 웹 서버에서 미니 Redis에 붙어서:

로그인 세션 저장
게시글 조회수 증가
인기글 캐시 저장
이 로직을 처리합니다.
웹페이지를 만든다
HTML/JS로 로그인, 게시글 열기, 인기글 갱신 버튼을 만들고,
개발자 콘솔에서 Redis 명령 흐름이 보이게 하면 발표용으로 좋습니다.

재시작 복구 테스트를 꼭 한다
이게 저장 기능의 핵심 검증입니다.
