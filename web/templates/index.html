MongoDB vs Redis 읽기 비교

무엇을 비교하나?

같은 게시글을 여러 번 읽어서 평균 시간을 봅니다. 원본 저장소와 메모리 캐시의 차이를 같은 조건에서 비교합니다.

MongoDB 경로

원본 게시글을 DB에서 읽기 영속 저장소에서 같은 데이터를 다시 읽습니다.

Redis 경로

같은 게시글을 캐시에서 읽기 메모리에 저장된 값을 바로 꺼냅니다.
게시글을 선택하면 클릭할 때마다 실시간 읽기 정보가 갱신됩니다.

첫 클릭은 DB read, 다음 클릭은 cache hit와 TTL 카운트다운을 보여줍니다.

저장소 구성 정보를 불러오는 중입니다.

게시글 원본은 DB, 캐시는 Redis 메모리에서 처리됩니다.

추천 시연 순서

  1. 같은 게시글을 한 번 눌러 DB READ가 찍히는지 봅니다.
  2. 바로 다시 눌러 CACHE READ와 SPEEDUP이 뜨는지 봅니다.
  3. TTL이 0초가 될 때까지 기다린 뒤 다시 눌러 DB READ로 돌아오는지 확인합니다.
  • 처음 화면이 열리면 서버 상태, 세션, 게시글 목록, 인기글 순서로 확인합니다.
  • 게시글을 누르면 조회수가 1 증가하고 인기글 캐시가 무효화됩니다.
  • TTL은 10초로 동작하며 만료되면 다시 DB에서 읽습니다.
  • 새로고침 후에도 세션 토큰이 남아 있으면 로그인 상태를 복구합니다.
