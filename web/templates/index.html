DB와 캐시 접근 속도 비교

캐시를 비운 뒤 첫 클릭은 DB만, 이후 cache hit 클릭은 CACHE만 표시합니다.

둘 다 함께 비교하려면 SQLite vs Cache 측정 버튼으로 실제 저장소 직접 조회와 메모리 키 직접 조회 평균을 동시에 계산합니다.

저장소 구성을 불러오는 중입니다.

캐시는 메모리 조회, 게시글은 로컬 디스크 기반 영속 저장소로 비교합니다.

  • 처음 화면이 열리면 서버 상태, 세션, 게시글 목록, 인기글을 순서대로 확인합니다.
  • 게시글을 열면 조회수가 1 증가하고 인기글 캐시는 무효화됩니다.
  • 캐시를 지운 뒤 첫 클릭은 DB 카드만, 이후 cache hit 클릭은 CACHE 카드만 갱신됩니다.
  • SQLite vs Cache 측정 버튼은 실제 저장소 직접 조회와 메모리 직접 조회 평균을 함께 비교합니다.
  • 새로고침 뒤에도 토큰이 살아 있으면 로그인 상태를 복구합니다.
