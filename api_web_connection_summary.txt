API와 웹페이지 연결 정리

1. 연결한 라우트
- GET /health
  : 페이지가 처음 열릴 때 서버가 정상 동작하는지 확인합니다.
- GET /posts/{post_id}
  : 게시글 상세 보기와 게시글 목록 새로고침에 사용합니다.
- PUT /posts/{post_id}
  : 상세 보기에서 제목, 내용, 작성자를 수정 저장할 때 사용합니다.

2. 웹페이지에서 연결한 위치
- app.js
  : 페이지 시작 시 /health 호출
  : 목록 새로고침 버튼 클릭 시 각 게시글 id로 GET /posts/{id} 호출
  : 게시글 열기 버튼 클릭 시 GET /posts/{id} 호출
  : 상세 패널의 수정 저장 버튼 클릭 시 PUT /posts/{id} 호출

3. 현재 아직 없는 라우트
- /login
- /logout
- /top-posts
- /posts (전체 목록 한 번에 가져오는 라우트)

4. 지금 동작 방식
- 로그인/로그아웃 버튼은 아직 실제 API 연결이 아니라 화면 상태만 임시 변경합니다.
- 게시글 목록은 현재 화면에 들어 있는 post id(1, 2, 3)를 기준으로 각각 GET 요청을 보내서 만듭니다.
- 인기글 영역은 별도 API가 없어서 현재 불러온 게시글 데이터를 바탕으로 화면에 표시합니다.

5. 다음에 추가하면 좋은 것
- GET /posts 라우트 추가
- GET /top-posts 라우트 추가
- POST /login, POST /logout 라우트 추가
- 조회수 증가용 POST /posts/{id}/view 라우트 추가
