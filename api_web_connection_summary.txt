API와 웹페이지 연결 정리

1. 연결한 라우트
- GET /health
  : 페이지가 처음 열릴 때 서버가 정상 동작하는지 확인합니다.
- GET /posts
  : 전체 게시글 목록을 한 번에 불러옵니다.
- GET /posts/{post_id}
  : 특정 게시글 상세를 읽어옵니다.
- PUT /posts/{post_id}
  : 상세 보기에서 제목, 내용, 작성자를 수정 저장할 때 사용합니다.
- GET /top-posts
  : 인기글 상위 3개를 가져옵니다.
- POST /login
  : 사용자 이름으로 간단한 세션을 만듭니다.
- POST /logout
  : 전달한 토큰 기준으로 세션을 삭제합니다.
- POST /posts/{post_id}/view
  : 게시글을 열 때 조회수를 1 증가시킵니다.

2. 웹페이지에서 연결한 위치
- app.js
  : 페이지 시작 시 /health 호출
  : 페이지 시작 시 GET /posts, GET /top-posts 호출
  : 목록 새로고침 버튼 클릭 시 GET /posts, GET /top-posts 호출
  : 게시글 열기 버튼 클릭 시 POST /posts/{id}/view 호출
  : 상세 패널의 수정 저장 버튼 클릭 시 PUT /posts/{id} 호출
  : 로그인 버튼 클릭 시 POST /login 호출
  : 로그아웃 버튼 클릭 시 POST /logout 호출

3. 지금 동작 방식
- 게시글 목록은 GET /posts 로 전체를 한 번에 불러옵니다.
- 인기글은 GET /top-posts 로 가져오고, MiniRedis 캐시를 사용합니다.
- 게시글 열기는 POST /posts/{id}/view 로 처리해서 조회수를 함께 올립니다.
- 게시글 수정 후에는 목록과 인기글을 다시 불러와 화면을 갱신합니다.
- 로그인/로그아웃도 실제 API 요청으로 연결되어 있습니다.

4. 참고
- 로그인은 간단한 데모용 세션입니다.
- 인기글 정렬은 현재 조회수 기준입니다.
- 조회수와 캐시 상태는 MiniRedis를 통해 관리합니다.
